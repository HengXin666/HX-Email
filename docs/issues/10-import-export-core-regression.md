# 补齐导入导出和旧项目核心回归能力

## What to build

在新架构下补齐旧版 outlookEmailPlus 的核心导入导出和回归能力，确保用户可以迁移、备份并验证第一阶段核心功能。该 issue 不要求恢复后置能力，如浏览器扩展、完整开放 API、通知、一键更新、AI 增强和插件化临时邮箱。

## Acceptance criteria

- [ ] 支持导入和导出邮箱账号、可用邮箱、别名、平台、平台绑定、分组和标签的核心数据。
- [ ] 导入时遵守用户内唯一约束和多用户隔离。
- [ ] 提供旧项目核心能力的回归测试清单或自动化测试。
- [ ] README 或迁移文档说明从旧版参考数据迁移到新模型的口径。
- [ ] 明确列出第一阶段后置能力，避免误认为已完成完整旧项目复刻。
