# 管理平台并将平台绑定到可用邮箱

## What to build

实现平台和平台绑定。用户可以创建平台，在可用邮箱详情中绑定平台，并维护绑定状态和备注。系统可以提供邮件解析得到的平台候选，但必须由用户确认后才创建平台或绑定。

## Acceptance criteria

- [ ] 用户可以创建、编辑、搜索自己的平台。
- [ ] 平台名称在同一用户内唯一且大小写敏感。
- [ ] 用户可以把一个可用邮箱绑定到多个平台，一个平台也可以绑定多个可用邮箱。
- [ ] 同一用户下同一组可用邮箱和平台只能有一条绑定记录。
- [ ] 平台绑定支持 `active`、`pending_verification`、`risk`、`disabled`、`archived` 状态。
- [ ] 平台绑定支持备注。
- [ ] 邮件解析只产生平台候选，不自动创建平台或绑定。
